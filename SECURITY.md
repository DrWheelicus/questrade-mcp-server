# Security Policy

## Supported Versions

Security fixes are applied to the latest code on `main`.

## Reporting a Vulnerability

Do not disclose vulnerabilities in public issues or pull requests.

Report privately to the repository owner with:

- A clear description of the vulnerability
- Steps to reproduce or proof of concept
- Expected impact and severity
- Suggested remediation (if available)

Please include sanitized logs only. Never share live tokens, API credentials,
or sensitive account information.

## Disclosure Process

After receiving a report, maintainers will:

1. Confirm receipt
2. Reproduce and assess impact
3. Prepare and validate a fix
4. Coordinate disclosure timing with the reporter
